Detailed Description

A structure to keep the command line options for an AlignStats instance This structure is used to keep the constructors very simple.

Member Data Documentation

string AlignStats::options::align_summary_file

a string representing the absolute path for the alignment summary error table

double AlignStats::options::align_summary_filter_accuracy

a double representing the % of the read that must be error free in order to be considered for the alignment summary error table. This filter value doesn't effect the alignment.summary or sam.parsed output. Filtering formula: if (((align_summary_filter_len - cummaltive_errors_in_read) / align_summary_filter_len) > align_summary_filter_accuracy)

int AlignStats::options::align_summary_filter_len

an int represting the minimum lenght a read must be in order to be considered for the alignment summary error table. This length is also used for the alignment.summary file and sam.parsed

int AlignStats::options::align_summary_len_step

an int representing the interval of read lengths between the min and max lengths in the alignment summary table

int AlignStats::options::align_summary_max_errors

an int representing the maximum number of errors to be output to the table. one column is produced for each +1 value until the value stored in align_summary_max_errors. For instance, if this is 3 the columns output would be: err1 err2 err3+ The final column always has a trailing + character. This indicates that reads in this column have at least that number of errors

int AlignStats::options::align_summary_max_len

an int representing the maximum length for a read to be considered in the alignment summary error table

int AlignStats::options::align_summary_min_len

an int representing the minimum length for a read to be considered in the alignment summary error table

string AlignStats::options::bam_file

a string representing the absolute path to a bam file, or just a file name

string AlignStats::options::bam_index

a string representing the absolute path to a bam file index

int AlignStats::options::buffer_size

an integer representing the number of reads that is assigned to each thread for a work load. If coverage is being calculated the buffer size is only approximately this size. It can be slightly larger, or smaller

bool AlignStats::options::debug_flag

a bool that will print debug messages to stderr. very verbose, be careful

long AlignStats::options::filter_length

the minimum length a read must be to be included in calculations and output [Default: 20]

string AlignStats::options::genome_info

a string representing the absolute path to a genome.info.txt file (internal Ion Torrent file format [Default: ""]

bool AlignStats::options::help_flag

a bool to print the help message to stdout. exits the program if true

bool AlignStats::options::iupac_flag

a bool flag that will check the MD tag for IUPAC codes and see if they're actually errors to the reference. use this if your reference contains IUPAC codes instead of bases in some positions

bool AlignStats::options::keep_iupac

a bool flag that will put iupac codes into the sam.parsed file. only has an effect if iupac_flag == true

std::string AlignStats::options::list_of_files

a string representing the absolute path to a file

int AlignStats::options::max_coverage

an int representing the maximum coverage that is allowed for 1 position in the genome if a particular position has > max_coverage reads aligning to it, only the first N reads are used, where N = max_coverage. [Default: 20000]

int AlignStats::options::num_threads

an integer representing the number of threads the program will try to use

string AlignStats::options::out_file

a string representing the absolute path to the prefix of output file names [Default: "Default"]

string AlignStats::options::q_scores

a comma seperate string of integers reprsenting the phred values to use for the alignment.summary output

int AlignStats::options::sam_parsed_flag

a flag to signal whether or not to create the sam.parsed file (can be very large file)

long AlignStats::options::sample_size

a long representing the number of reads to sample

bool AlignStats::options::skip_cov_flag

a bool that allows the user to skip coverage calculations. this speeds up the program considerably, and reduces memory use

long AlignStats::options::start_slop

an integer representing the first N bases to be ignored when considering errors in an alignment

bool AlignStats::options::stdin_bam_flag

a bool representing whether or not the input is from stdin, and binary input

bool AlignStats::options::stdin_sam_flag

a bool representing whether or not the input is from stdin, and raw text input

long AlignStats::options::total_reads

a long to represent the total number of reads being input. this is used for sampling purposes in order to extrapolate stats

bool AlignStats::options::truncate_soft_clipped

a bool to represent whether or not to ignore soft clipped bases in the alignment. This can really effect reads which map to the reverse strand and have a # of leading soft clipped bases in the alignment [Default: true]
